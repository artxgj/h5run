According to wikipedia, the Tabularium was the official records office of ancient Rome. Hi! My name's Art. Cloud Tabularium is a place for me to learn about developing prototype applications that store records in the Google cloud.

My first application is an iPhone 3G web application that records the time and distance of runs or walks using GPS. This is accomplished through using iPhone Safari's W3C Geolocation API. The GPS data are stored in real-time in a Google App Engine datastore and used by the "Run Stats & Maps" to generate a map of the course and the split times. To the right is an image of a recorded run. I am using the iPhone run recorder to help me keep track of my long runs and training for a marathon.

iPhone-runners and walkers are very welcome to try my run recorder prototype. In theory, the run recorder should work in any browser that supports the W3C Geolocation API. I haven't had time to search the web for mobile browsers that support this API. Both run recorder and "Run Maps & Stats" rely on Google to authenticate its users. As such anyone who wants to try these programs must have a gmail account. At the moment, there is no sharing capability. Therefore users of this program can only see their own run data. Feedback is very much welcomed, please send them to beijingbayou email address listed above.

An explanation of the code that generates the maps is available at my Beijing Bayou website.

Caveats

Using the iPhone's GPS will drain the iPhone's battery life. The longest I've been able record my run is a little over 2 hours; the battery was fully drained. Aside from this issue, the precision of the GPS distance calculations is dependent on the qualiy of the iPhone's cell reception.

Checklist

Make sure these iPhone settings are configured before using the run recorder:

  • Location Services is on.
  • Auto-Lock is Never
  • Passcode Lock is Off
